# Руководство для контрибьюторов

## Как начать

1. Форкните репозиторий и создайте новую ветку от `main`.
2. Убедитесь, что установлены Node.js 24+ LTS, PostgreSQL и зависимости (`npm install`).
3. Настройте `.env` и примените миграции (`npx prisma migrate deploy`).

## Стиль кода

- TypeScript: следуйте настройкам `tsconfig.json`, избегайте `any` без явной необходимости.
- Импорт путей: в исходниках оставляйте расширение `.js`, чтобы сборка ESM работала в NodeNext.
- Markdown: придерживайтесь структуры разделов, используйте русскоязычные заголовки.
- Комментарии: добавляйте только для сложной логики или нетривиальных решений.

## Тесты и проверка

Перед отправкой PR запустите:

```powershell
npm run lint
npm run test
npx tsc --noEmit
```

> ⚠️ **Важно:** PR с падающими тестами или ошибками типизации не принимаются.

## Коммиты

- Используйте осмысленные сообщения в стиле `feat:`, `fix:`, `docs:`, `chore:`.
- Группируйте изменения по смыслу, избегайте больших «всё-в-одном» коммитов.

## Оформление Pull Request

- Кратко опишите мотивацию изменений и затронутые областя.
- Добавьте шаги для тестирования (как ручного, так и автоматического).
- Приложите скриншоты UI-изменений (бот, веб-панель).

Спасибо за вклад в развитие проекта!
